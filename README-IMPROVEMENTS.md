## Техническа документация

### Валидация с Zod

Имплементирали сме Zod схеми за всички основни типове данни в системата. Пример:

```typescript
export const invoiceSchema = z.object({
  clientId: z.string().min(1, "Клиентът е задължителен"),
  companyId: z.string().min(1, "Фирмата е задължителна"),
  issueDate: z.string().refine(str => !isNaN(Date.parse(str)), "Невалидна дата"),
  dueDate: z.string().refine(str => !isNaN(Date.parse(str)), "Невалидна дата"),
  status: z.enum(['DRAFT', 'UNPAID', 'PAID', 'OVERDUE', 'CANCELLED']).default('DRAFT'),
  items: z.array(invoiceItemSchema).min(1, "Фактурата трябва да има поне един артикул"),
  notes: z.string().optional(),
  termsAndConditions: z.string().optional(),
}).refine(data => {
  const issueDate = new Date(data.issueDate);
  const dueDate = new Date(data.dueDate);
  return dueDate >= issueDate;
}, {
  message: "Падежната дата трябва да бъде след датата на издаване",
  path: ["dueDate"],
});
```

### Тестване

Препоръчваме следния подход за писане на тестове:

1. **Unit тестове**: За компоненти с бизнес логика, utilities, hooks
2. **Integration тестове**: За форми и сложни компоненти
3. **E2E тестове**: За критични потребителски пътеки (с Playwright или Cypress)

Пример за модерен тест:

```typescript
describe('ValidatedClientForm', () => {
  it('should validate required fields', async () => {
    const user = userEvent.setup();
    render(<ValidatedClientForm />);
    
    await user.click(screen.getByRole('button', { name: /запази/i }));
    
    await waitFor(() => {
      expect(screen.getByText(/името е задължително/i)).toBeInTheDocument();
    });
  });
});
```
